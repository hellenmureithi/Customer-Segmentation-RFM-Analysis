# Customer Segmentation Using RFM Analysis and K-Means Clustering

## Overview

This project performs customer segmentation using **RFM Analysis (Recency, Frequency, Monetary)** combined with **K-Means Clustering** to group customers based on purchasing behavior.

The analysis is conducted on the **Online Retail Dataset** and aims to help businesses:

* Identify high-value customers.
* Detect customers at risk of churn.
* Develop targeted marketing campaigns.
* Optimize customer retention strategies.
* Increase revenue through personalized offers.

This project demonstrates a complete end-to-end data science workflow:

1. Data cleaning and preprocessing.
2. Feature engineering.
3. RFM metric calculation.
4. Customer clustering.
5. Segment interpretation.
6. Visualization.
7. Business recommendations.

## RFM Analysis Explained

RFM analysis evaluates customer behavior using three metrics.

### Recency (R)

Number of days since a customer's last purchase.

* Lower recency = better.
* Indicates how recently the customer was active.

### Frequency (F)

Number of unique transactions made by the customer.

* Higher frequency = better.
* Indicates engagement and loyalty.

### Monetary (M)

Total amount spent by the customer.

* Higher monetary value = better.
* Indicates customer value.

## Data Cleaning Process

The following preprocessing steps were performed.

### 1. Missing Values

Removed rows with missing:

* `CustomerID`
* `Description`

### 2. Returned Transactions

Transactions with negative quantities were excluded because they represent returns or cancellations.

### 3. Duplicate Records

Duplicate rows were removed.

### 4. Invalid Prices

Rows with `UnitPrice <= 0` were removed.

### 5. Date Conversion

`InvoiceDate` was converted to datetime format.

### 6. Feature Engineering

Created:

* `TotalPrice = Quantity × UnitPrice`

---

## RFM Metric Creation

Customer-level metrics were generated by grouping by `CustomerID`.

### Recency

```python
snapshot_date = df_cleaned['InvoiceDate'].max()
rfm['Recency'] = (snapshot_date - last_purchase_date).days
```

### Frequency

Count of unique invoices.

### Monetary

Sum of `TotalPrice`.

---

## Machine Learning Approach

### Feature Scaling

RFM metrics were standardized using `StandardScaler`.

### Clustering Algorithm

K-Means clustering was applied with:

```python
KMeans(n_clusters=4, random_state=42)
```

### Pipeline

A Scikit-learn pipeline ensures reproducibility.

```python
Pipeline([
    ('scaler', StandardScaler()),
    ('kmeans', KMeans(n_clusters=4, random_state=42))
])
```

## Segment Profiles

### VIP Customers

Characteristics:

* Very recent purchases.
* Extremely high transaction frequency.
* Highest total spending.

Business Actions:

* Loyalty rewards.
* Exclusive discounts.
* Early access to products.
* Premium customer support.

---

### Loyal Customers

Characteristics:

* Regular purchases.
* Strong monetary contribution.

Business Actions:

* Upselling.
* Cross-selling.
* Referral incentives.

---

### Potential Loyal Customers

Characteristics:

* Moderate frequency.
* Moderate spending.

Business Actions:

* Personalized offers.
* Product recommendations.
* Nurture campaigns.

---

### At Risk Customers

Characteristics:

* Long time since last purchase.
* Low frequency.
* Low spending.

Business Actions:

* Win-back campaigns.
* Time-limited discounts.
* Re-engagement emails.
